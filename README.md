### Microservices Overview

This project follows a microservices architecture with the following components:

**Application Layer:**
- **Vitrine Web** (Port 8080) - Main web application built with Spring Boot

**Service Layer:**
- **Service CRUD** (Port 8081) - RESTful API for data management
- **Service IA** (Port 5000) - AI/ML recommendation engine (Flask)
- **Keycloak** (Port 8180) - Identity and access management server

**Data Layer:**
- **MySQL** (Port 3306) - Relational database

### Architecture Flow

1. Users access the web interface through **Vitrine Web** (8080)
2. **Vitrine Web** authenticates users via **Keycloak** (8180)
3. Data operations are handled by **Service CRUD** (8081)
4. AI recommendations are generated by **Service IA** (5000)
5. All data is persisted in **MySQL** database (3306)

ğŸš€ Installation
1. Clone the Repository
git clone https://github.com/mehdilounnas/voyage-intelligence.git
cd voyage-intelligence
2. Database Setup

Start XAMPP and create database:
CREATE DATABASE tourism_db;
Tables will be created automatically by Hibernate.
3. Keycloak Configuration

Start Keycloak:
cd keycloak-26.4.5/bin
./kc.bat start-dev --http-port=8180
Configure:

    Access: http://localhost:8180/admin (admin/admin)

    Create realm: tourism-realm

    Create client: tourism-app

    Set redirect URI: http://localhost:8080/*

    Create users: admin/admin123

4. Configure Application

service-crud/src/main/resources/application.properties:
server.port=8081
spring.datasource.url=jdbc:mysql://localhost:3306/tourism_db
spring.datasource.username=root
spring.datasource.password=
vitrine-web/src/main/resources/application.properties:

server.port=8080
spring.security.oauth2.client.registration.keycloak.client-id=tourism-app
spring.security.oauth2.client.registration.keycloak.client-secret=YOUR_SECRET
spring.security.oauth2.client.provider.keycloak.issuer-uri=http://localhost:8180/realms/tourism-realm
5. Install Python Dependencies
cd service-ia
pip install -r requirements.txt
